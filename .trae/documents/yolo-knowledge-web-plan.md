# YOLO使用和优化 - 知识库网页版制作计划

## 📋 项目概述

将 `yolo使用和优化` 文件夹中的 **15篇Markdown笔记** 转换为一个精美的、可交互的静态网页知识库。

### 内容清单（15个文件）

| 分类 | 文件数 | 文件列表 |
|------|--------|----------|
| 首页 | 1 | `00-首页.md` |
| YOLO基础概念 | 3 | `YOLOv8架构详解`, `YOLO发展历程`, `目标检测基础` |
| Ultralytics框架入门 | 4 | `快速开始指南`, `核心API详解`, `环境搭建与安装`, `配置文件说明` |
| 实战应用 | 2 | `数据集准备与格式转换`, `模型训练完整流程` |
| 性能优化 | 3 | `推理速度优化`, `模型压缩技术`, `训练加速策略` |
| 资源与参考 | 2 | `学习路线图`, `推荐论文列表` |

### 内容特点分析
- **Mermaid图表**: 流程图、时序图、架构图等（每个文件都有）
- **代码块**: Python、YAML、Bash、Dockerfile等，大量代码示例
- **数学公式**: LaTeX格式（IoU、损失函数等）
- **表格**: 性能对比、参数说明等
- **内部链接**: Obsidian格式的 `[[链接]]`
- **Emoji图标**: 大量用于标题和导航

---

## 🎨 设计方向

### 美学定位: **"现代技术文档 + 终端黑客美学"**

- **风格灵感**: 类似 Vercel 文档 / Linear 文档的现代感，但融入 AI/ML 技术的科技感
- **配色方案**: 深色主题为主，搭配霓虹绿/青色作为强调色（呼应 YOLO 的检测框视觉）
- **字体选择**: 
  - 标题: `JetBrains Mono` 或 `Space Mono`（代码/技术感）
  - 正文: `Inter` 替代品如 `Plus Jakarta Sans`（避免泛用）
- **特色元素**: 
  - 代码块风格的侧边栏
  - 渐变光晕背景（模拟神经网络激活图）
  - 卡片式内容布局带微妙玻璃态

---

## 🏗️ 技术架构

### 方案选择: **纯静态 HTML/CSS/JS（单页应用）**

不使用 React/Vue 等框架，原因：
1. 内容型网站，无需复杂状态管理
2. 直接部署到 GitHub Pages / Vercel / Netlify
3. 加载速度极快
4. 无需构建步骤

### 核心技术栈

| 功能 | 技术选型 | 说明 |
|------|----------|------|
| Markdown解析 | `marked.js` | 高性能，支持扩展 |
| 代码高亮 | `highlight.js` | 支持 Python/YAML/Bash 等 |
| Mermaid图表 | `mermaid.js` | 渲染流程图/架构图 |
| 数学公式 | `KaTeX` | 比 MathJax 更快 |
| 搜索功能 | `flexsearch.js` | 轻量级全文搜索 |
| 图标 | 内联 SVG | 自定义检测框风格图标 |

## 🔧 实施步骤

### Step 1: 项目初始化
- 创建项目目录结构
- 准备所有静态资源
- 建立 HTML 入口文件

### Step 2: 构建 Markdown 解析管线
- 编写自定义 marked.js 扩展处理器
- 处理代码块语言标识和行号
- 处理 `[[wiki-link]]` 格式的内部链接
- 处理 Emoji 和特殊标记

### Step 3: 实现页面布局系统
- CSS Grid/Flexbox 响应式布局
- 侧边栏导航组件（支持折叠）
- 固定头部导航栏
- 内容区域滚动样式

### Step 4: 内容渲染引擎
- Mermaid 图表自动渲染（带加载动画）
- KaTeX 数学公式渲染
- highlight.js 代码高亮（YOLO定制主题）
- 表格响应式处理
- 图片懒加载

### Step 5: 交互功能实现
- 全文搜索（即时搜索，高亮关键词）
- 主题切换（深色/浅色）
- 侧边栏折叠/展开
- 文章目录（TOC）自动生成+点击跳转
- 阅读进度指示器
- 键盘快捷键（`?`显示帮助）

### Step 6: 视觉打磨
- CSS 动画（页面切换、元素入场）
- 渐变背景效果
- 代码块复制按钮
- 外链图标识别
- 打印友好样式

### Step 7: 数据集成
- 将15个 .md 文件转换为内嵌 JSON 数据
- 构建导航树结构
- 生成首页聚合卡片
